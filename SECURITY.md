# Security Policy

We take security issues seriously. If you discover a vulnerability in the **NULL_POINTER** project, please report it privately using the guidelines below to help keep our simulation playgrounds secure.

---

## Supported Versions

Only the latest release version on the `main` branch receives active security updates and patch updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you identify a security flaw or vulnerability (such as AST sandbox breakouts, authentication bypass, or arbitrary command execution flaws):
1. Draft an email detailing the vulnerability.
2. Include a Proof of Concept (PoC) script or detailed replication steps to help us reproduce the issue.
3. Send your report to: **security@nullpointer-labs.org** (Note: Mock contact for playground purposes).

We will verify and acknowledge receipt of your report within **48 hours** and provide a timeline for resolution.

## Vulnerability Response SLA

* **Initial Acknowledgement**: Within 2 business days.
* **Triage & Validation**: Within 5 business days of receipt.
* **Vulnerability Fix**: We aim to release security patches for verified high-severity flaws within 30 days.
* **Public Disclosure**: Coordinated public disclosure will happen only after an official patch is published.
